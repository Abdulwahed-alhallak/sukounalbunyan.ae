# 👥 شرح متعمق لنظام الموارد البشرية (HRM Module Deep Dive)

بما أن التركيز الحالي يتجه نحو بيئة الموارد البشرية (HRM)، قمنا بتخصيص هذا الدليل المستقل لشرح التدفق المعماري والبرمجي لوحدة الـ HRM داخل `packages/noble/Hrm`.

## 1. المعمارية ودورة حياة الموظف (Employee Lifecycle)
يقوم موديول الموارد البشرية بالتعامل مع عدة كيانات تشكل معاً مسار الموظف بالمؤسسة:
1. **الأقسام الأكاديمية والمؤسسية (Departments):** هيكلية الموظفين. *ملاحظة: تم توحيد المصطلحات في المنصة ليتم استخدام Department (القسم) بدلاً من مصطلح Specialty (الاختصاص) القديم لضمان الاتساق الإداري والاكاديمي.*
2. **بروفايل الموظف (Employee Profile):** ويُدار بشكل رئيسي عبر `EmployeeController`. يتضمن البيانات الشخصية، البنكية، والمستندات.
3. **الحضور والانصراف (Attendance/Timesheet):** تتبع وقت القدوم والمغادرة.
4. **الرواتب والإجازات (Payroll & Leaves):** معالجة السلف والإصدارت المالية للموظف، وتصريف أنواع الإجازات.

## 2. متحكم الموظفين (EmployeeController)
يتعامل `EmployeeController.php` مع الوظائف التالية:
- الاستعلام عن قائمة الموظفين (Index) مع تطبيق صلاحيات `company_id`.
- تخزين بيانات الموظف (Store) عبر نماذج (Form Requests) للتحقق من خلو الأخطاء وإنشاء حساب مستخدم مرتبط في نظام Laravel الأساسي (جدول `users`).
- إدارة التعيينات (Designation) التي تحدد الراتب الأساسي وساعات العمل الافتراضية.

### مسألة حساسة: ربط بيانات المستخدم بالموظف
في نظامنا، كل موظف هو في الواقع `User` بداخل النظام لكي يمكنه تسجيل الدخول واستلام المهام. 
عند إنشائك لموظف جديد، النظام يقوم بعملية مزدوجة:
```php
// كود توضيحي لعملية الحفظ
$user = User::create([...]); // إنشاء حساب الدخول
$user->assignRole('Staff');  // إعطاؤه صلاحيات
$employee = Employee::create([ 
    'user_id' => $user->id, ... 
]); // ربط حساب الدخول ببطاقة الموظف
```

## 3. مسارات الموارد البشرية (`Routes/web.php`)
يحتوي ملف توصيل مسارات الرهط لـ HRM عادة على:
- مسارات الموظفين `/employee`، المسميات الوظيفية `/designation`، الإدارات `/department` (والتي حلت محل التخصصات).
- مسارات سياسات الشركة واللوائح `/company-policy`.
- كافة المسارات محاطة داخل Middleware:
  `['auth', 'PlanModuleCheck:Hrm']` للتأكد بأن الشركة قد اشتركت في خدمة الموارد البشرية.

## 4. التكامل مع واجهة المستخدم (Frontend - React/Inertia)
عند طلب قائمة الموظفين، يرسل السيرفر كائناً مجدولاً إلى:
`Inertia::render('Hrm/Employee/Index', [ 'employees' => $employees ])`
حيث يستقبل ملف `Index.tsx` هذه المصفوفة ويعرضها بداخل `DataGrid` المخصص لعرض البيانات مع مميزات البحث والفرز، وهي كلها مبنية على قوانين Vercel الثابتة والتصميم الأنيق للمنظومة.
