# 🎨 قواعد التصميم والثيم المرئي (Theme & UI/UX Guidelines)

يعتبر المظهر المرئي جزءاً ثابتاً غير قابل للتجاوز في بناء أي صفحة أو وحدة جديدة. تطبيق Noble Architecture مبني على مكتبة Tailwind CSS ولكن بتخصيصات تعتمد على (Design Tokens) من Vercel وتأثيرات الـ Glassmorphism.

## 1. قواعد الخطوط (Typography)
تم بناء الواجهات لخدمة الجانبين اللاتيني والعربي بشكل ممتاز:
- **للغات اللاتينية (الإنجليزية):** يستخدم خط `Geist Sans`.
- **للغة العربية (RTL):** يستخدم خط `IBM Plex Sans Arabic` ليعطي طابعاً رسمياً مقروءاً بوضوح.
- **للأكواد والأرقام (Monospace):** يستخدم خط `Geist Mono`.
*ملاحظة: يحظر استخدام أي خطوط خارجية أخرى أو الاعتماد على الخطوط الافتراضية للمتصفح إلا كحل فشل متأخر (Fallback).*

## 2. إدارة الألوان (Color System)
النظام يمنع تماماً استخدام الألوان الافتراضية عبر Hex أو أسماء Tailwind المباشرة مثل `text-red-500` أو `bg-blue-600` في الأكواد مباشرة. بل يجب المرور عبر متغيرات `app.css` الدلالية:
- **الخلفيات:** `bg-background`, `bg-card`, `bg-popover`.
- **النصوص والتحذيرات:** `text-foreground` و `text-muted-foreground` و `text-destructive`.
- **الأزرار الأساسية الرئيسية:** يجب أن تستخدم `bg-primary text-primary-foreground`.
- **التأثيرات الجمالية الخاصة (Vercel Magic Tokens):** تُستخدم خطوط مضيئة وتأثيرات طفيفة بمتغيرات مثل `var(--geist-cyan)`.

## 3. الفئات والهياكل البنائية (Structural Elements)
تم توحيد هياكل الكروت والمدخلات من خلال فئات عامة ومكونات (Components) مجهزة مسبقاً:
- **`.vercel-card`**: المكون الأساسي الموحد لإعطاء حواف مرسومة وتأثير ظل خفيف عند التمرير (Hover).
- **`.premium-card`**: تأثير يستخدم لواجهات ومؤشرات الأداء (KPIs) ليعطي طابع (Glassmorphism).
- **المدخلات والاختيارات:** يُمنع استخدام عناصر HTML الافتراضية مثل `<select>`. يجب استخدام المكونات المخصصة مثل `<Combobox>` الذي يدعم البحث المتقدم ويواكب لغة التصميم (Vercel-inspired).

- **ممنوع كتابة نصف قطر الزاوية (Border Radius) بشكل عشوائي:** يجب الالتزام بـ `rounded-sm` أو `rounded-md` أو `rounded-lg` فقط للحفاظ على وحدة الشكل.

## 4. تأثير الزجاج الشفاف (Glassmorphism)
يُعتمد بشكل كبير في الواجهات الليلية (Dark Mode)، القوائم الجانبية والنوافذ المنبثقة (Modals & Dialogs).
- يجب التوفيق بين `bg-background/80` و `backdrop-blur-md` أو `backdrop-blur-xl`.
- ممنوع تماماً استخدام خلفية سوداء صماء بلا تأثير ضبابي على الطبقات العلوية للنظام.

## 5. دعم الاتجاهات والعربية (RTL Configuration)
لكي يدعم النظام التوطين والترجمة بدون قلب التصميم بأكمله عبر CSS جديد، تم الاعتماد على خصائص المنطقية (Logical Properties) الخاصة بـ Tailwind.
- **القاعدة الذهبية:** تجنب نهائياً استخدام المسافات بناءً على الاتجاه المادي (مثل `ml-4` أو `pr-2` أو `left-0`).
- **البديل المتوافق:** استخدم المسافات المنطقية: `ms-4` (بديل Margin Left)، `pe-2` (بديل Padding Right)، `start-0` ، `end-0`.
