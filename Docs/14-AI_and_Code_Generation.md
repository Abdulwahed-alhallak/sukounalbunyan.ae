# 🤖 الذكاء الاصطناعي والمساعد الذكي (AI Assistant Module)

يمثل المساعد الذكي جزءاً مهماً من المنظومة، حيث تم تخصيص (Module) مخصص بمسار `packages/noble/AIAssistant`. في هذا الدليل نوضح كيفية تفعيل واستخدام محركات الذكاء الاصطناعي بالمنصة.

## 1. بنية الربط بأساسيات النماذج (Model Integrations)
يتيح موديول الذكاء الاصطناعي الربط المباشر مع مزودي الخدمات الكبرى لإنشاء النصوص واقتراح المحتوى للمستخدمين:
- يعتمد النظام عادةً على مفتاح (API Key) من منصة **OpenAI (ChatGPT)** أو مزودين آخرين.
- يتم إعداد المفتاح بشكل مركزي وتشفيره في إعدادات المنصة، ولا يطلع عليه الموظفون العاديون.

## 2. وظائف الذكاء الاصطناعي في المنصة
يستطيع النظام التدخل لمساعدة موظفي الشركة (المستأجر) في عدة مراحل:
- **تحسين الوصف والمدخلات:** مساعدة مسؤول المبيعات في كتابة وصف الفاتورة أو توليد ملامح لمنتج ما في موديول (ProductService).
- **الرد على التذاكر:** استخلاص إجابات مقترحة من سجلات الدعم والرد التلقائي على طلبات العملاء بداخل وحدة `SupportTicket`.
- **ترجمة المحتوى وتصحيحه:** تقديم نصوص لغوية سليمة في نظام الرسائل (Messenger) أو عند بناء قوالب الإيميل.
- **التخطيط الآلي:** في موديول `Taskly` يمكن استخدام الذكاء الاصطناعي لإنشاء وتوزيع المهام بناءً على عنوان المشروع.

## 3. التعامل البرمجي مع استدعاءات الذكاء الاصطناعي
يُدار الذكاء الاصطناعي عبر واجهة اتصال موحدة (Interface/Service) داخل كود المنظومة كي لا يتم كتابة أكواد الـ API العارية في الـ Controllers:
- يتم تجميع محتوى الطلب (Prompt) وتمرير اسم الـ `Model` (مثل `gpt-4o` أو `gpt-3.5-turbo`).
- يستقبل الخادم الاستجابة ويعيدها للواجهة الأمامية كـ `JSON Response`.
- الواجهة تعرض المحتوى بداخل (Rich Text Editor) أو في منطقة محددة ليتمكن الموظف من تعديله قبل اعتماده النهائي.
