# 🛡️ دليل الأمان وتعدد المستأجرين (Security & Multi-Tenancy)

يُعتبر الأمان والفصل التام للبيانات (Data Isolation) بين الشركات المشتركة (Tenants) من أهم ركائز نظام SaaS. في هذا المجلد نشير إلى المعايير الأمنية والهيكل المتبع لحماية المعلومات في المنصة.

## 1. مفهوم المستأجر (Tenant) في النظام

لا يُستخدم في هذا النظام قاعدة بيانات منفصلة لكل شركة (Database-per-Tenant) بل يتم الاعتماد على بنية قاعدة بيانات واحدة مشتركة (Single Database, Multi-Tenant).

- كل مستخدم لديه مُعرِّف `company_id` أو يُنسب لمُنشئ الحساب (Creator/Owner).
- يتم عزل البيانات برمجياً عبر الـ Scopes الخاصة بـ Laravel. فعند استعلام المبيعات مثلاً `SalesInvoice::all()`، يعمل الـ Global Scope خفية بالخلفية لرد الفواتير الخاصة بالشركة المسجل الدخول باسمها فقط.

## 2. إدارة الأدوار والصلاحيات (Roles & Permissions)

تُستخدم حزمة `spatie/laravel-permission` كعمود فقري للتحكم بالوصول:

- **المالك الرئيسي (Super Admin):** له القدرة على إدارة النظام بأكمله، الباقات، ومفاتيح الدفع. لا يمكن لأي حساب شركة الوصول لصفحات الـ Super Admin.
- **مدير الشركة (Company Admin):** يُعتبر مالك مساحة العمل (Workspace Owner). يمكنه التحكم بكامل الموديولات داخل باقته فقط.
- **الموظفون (Staff / Users):** يتم منحهم أدواراً (Roles) مخصصة من قبل الـ Company Admin ليتمكنوا من معالجة الفواتير، الحضور والانصراف، أو الموارد البشرية. الصلاحيات دقيقة جداً لكل وحدة.

## 3. التحقق والمصادقة (Authentication)

- يتم الاعتماد على `Laravel Sanctum` لحماية مسارات الـ API إن وجدت، وتمكين تسجيل الدخول الموثوق (Stateful Authentication) لتطبيق الصفحة الواحدة (SPA) المبني بـ Inertia.js.
- الـ CSRF Tokens مفعلة على كل استدعاء عبر `axios` المدمج مسبقاً في `resources/js/bootstrap.js`، لمحاربة ثغرات تزوير الطلبات عبر المواقع.

## 4. تدقيق الأمان وسجل النشاطات (Audit Logs & Security Audit)

تعد ميزة (Mission Command - Security Audit) إحدى الوحدات المتقدمة في المنصة:

- النظام يقوم بتسجيل (Log) دوري لمحاولات الدخول الفاشلة وتسجيل الدخول وتغييرات الصلاحيات وحفظها في قاعدة البيانات بجدول `audit_logs` أو وحدة التدقيق المركزية المخصصة لذلك.
- يمكن لمشرف النظام مراقبة الأنشطة المشبوهة، تتبع الآيبي (IP Address)، وإيقاف أي حساب يتعرض لمحاولات اختراق أو نشاط مريب فوراً.

## 5. تشفير البيانات الحساسة (Data Encryption)

أرقام البطاقات وكلمات المرور الخاصة بخدمات البريد (SMTP) أو مفاتيح الدفع (Stripe/PayPal) المعينة لكل شركة تُحفظ بشكل مشفر (Encrypted) داخل قاعدة البيانات باستخدام مفتاح التطبيق `APP_KEY` لتجنب تسريب بيانات الشركات حتى في حال حدوث تسريب لقاعدة البيانات المشتركة.
