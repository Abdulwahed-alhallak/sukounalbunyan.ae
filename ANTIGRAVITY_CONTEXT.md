# 🧠 Antigravity System Context (DionONE / Noble Architecture)

هذا الملف هو بمثابة "الذاكرة المرجعية" لنظام **Antigravity** (نظام الذكاء الاصطناعي). عند إعادة تشغيل النظام أو استئناف العمل، يجب على الذكاء الاصطناعي قراءة هذا الملف ليقوم باستعادة فهمه الكامل للمشروع ومساراته وتفاصيله بشكل فوري، دون الحاجة لسؤال المستخدم.

---

## 1. 🏗 هيكلية المشروع والمسارات (Project Structure & Paths)
يقع النظام التقني داخل المجلد الرئيسي `main-file/`:
- **الباك إند (Backend)**: يقع ضمن مجلدات Laravel القياسية مثل `app/` (لوجيك الأعمال، والـ Middleware)، و `routes/` (توجيه الـ web/api) و `config/`. بالإضافة إلى وحدات الـ SaaS المتعددة الموجودة داخل: `packages/noble/` (مثل وحدات Hrm, Taskly, Pos، إالخ).
- **الفرونت إند (Frontend)**: يقع بالكامل داخل مجلد `resources/js/`. يستخدم إضافة `Inertia.js` لربط React المتطور بمعمارية Laravel بسلاسة في صفحة واحدة (SPA).
- **الوحدات التنفيذية (Scripts)**: توجد ضمن مجلد `_scripts/` أدوات هامة مثل سكربت الرفع عبر (Hostinger SFTP).

## 2. 💻 التقنيات والإصدارات (Tech Stack & Versions)
- **الباك إند (Backend)**: 
  - إطار العمل: Laravel 12.0
  - لغة البرمجة: PHP 8.2+
  - قاعدة البيانات: MySQL (تتم إدارتها عبر الـ Eloquent ORM).
- **الفرونت إند (Frontend)**: 
  - إطار العمل: React 18.2.0 مدمجاً مع Inertia.js
  - بناء الواجهات: Vite 5.0
- **واجهة المستخدم والتصميم (UI & Styling)**: 
  - TailwindCSS 3.2.1
  - مكتبة المكونات المساعدة: Radix UI (Shadcn UI)
  - مؤثرات بصرية: Framer Motion
- **بوابات الدفع (Payment Methods)**: 
  - مدعومة في الكود عبر مكاتب مثل `stripe/stripe-php` و `srmklive/paypal`.

## 3. 🚀 طريقة الرفع والدفع على (GitHub Push)
لضمان حفظ الكود وتحديث النسخة الاحتياطية على GitHub بعد القيام بأي تعديلات هامة، ينفذ نظام الذكاء الاصطناعي أو المطور هذه الأوامر بالتسلسل:
```bash
git add .
git commit -m "رسالة توضيحية لما تم إنجازه، مثلا: Fix routing or update UI"
git push origin main
```
> **ملاحظة أمنية:** لا يجب أبداً إجراء Push لملف `.env` تجنباً لتسريب كلمات المرور والمفاتيح. تأكد أن الملف متواجد في `.gitignore`.

## 4. 🌍 طريقة التحديث والرفع للإنتاج على Hostinger (Deployment)
عملية نشر التحديثات على الخادم (Deploy) أوتوماتيكية تعتمد على سكربت خاص لتحديث النظام بدون توقفه. وتتم كالتالي:

1. **بناء الفرونت إند (Build)**:
   ```bash
   npm run build
   ```
2. يتم تجهيز وضغط جميع الملفات والتحديثات الخاصة ببيئة الإنتاج إلى أرشيف مضغوط باسم:
   `noble_production_build.tar.gz`
3. **تنفيذ سكربت الرفع (SFTP Pipeline)**:
   يتم تشغيل السكربت الموجود داخل `_scripts` باستخدام Node.js:
   ```bash
   node _scripts/hostinger_upload_sftp.js
   ```
   *تفاصيل السكربت:* يقوم هذا السكربت بعمل مسار آمن SSH إلى Hostinger (الآيبي `62.72.25.117` ومنفذ `65002` والمستخدم `u256167180`). ويرسل الملف المضغوط إلى الدليل `domains/noble.dion.sy/public_html`؛ بمجرد اكتمال الرفع يقوم بفك الضغط وتحديث النظام فورياً خلف الكواليس ثم يمسح ملف الأرشيف للتنظيف.

## 5. 👑 قواعد أساسية للذكاء الاصطناعي (Antigravity Core Directives)
- **إدارة تعدد الشركات (Multi-Tenancy)**: التطبيق يعمل بنظام شركات (SaaS). جميع الاستعلامات الداتا و جداول قاعدة البيانات يجب أن يتم التدقيق عليها باستخدام حقلي `company_id` و `created_by` كشرط أساسي لمنع تداخل بيانات عملاء النظام.
- **التصميم والأسلوب**: الاعتماد الافتراضي للواجهات يكون بوضعية (Dark Mode) وبمستوى (Premium Vercel-like CSS).
- **التوافق اللغوي**: تصميم الواجهات يجب أن يدعم الاتجاهين. لذلك لا نستخدم مسافات بتحديد الجهات كـ `ml-` أو `pr-`، وإنما الاتجاهات المنطقية `ms-` و `pe-` لتناسب اللغتين الانجليزية والعربية تلقائياً في Tailwind.
