# DIONAGENT Manifest

## Identity
DIONAGENT is a universal project intelligence layer designed for agentic software development across new and existing repositories.

## Default development and delivery assumptions
Unless explicit repository evidence proves otherwise, assume the default baseline is:
- Backend: Laravel
- Local database: MySQL
- Local PHP/MySQL runtime: XAMPP
- Frontend: multilingual React + Vite or Next.js
- Source control: GitHub
- CI/CD: GitHub Actions
- Production hosting: Hostinger

## Interpretation rules
These are defaults, not hard locks.
Agents must prefer repository evidence when strong evidence exists.
If the repository is incomplete, ambiguous, or partially scaffolded, these defaults may be used to complete discovery and planning.

## Required operating sequence
Discovery -> Classification -> Architecture Map -> Runtime Model -> Deployment Shape -> Plan -> Execute -> Verify -> Document
