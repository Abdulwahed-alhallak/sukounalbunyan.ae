# 🧩 دليل الإضافات والوحدات (Modules & Add-ons Architecture)

يُعتبر نظام (Sukoun Albunyan) نظاماً مبنياً على بنية الفسيفساء والموديولات المستقلة، بحيث يمكن تركيب أو إزالة ميزة دون أن يتأثر الكود الرئيسي للتطبيق (Core Base). هذا النهج مفيد جداً في نظام (SaaS) حيث يُستفاد منه لتشغيل الباقات المتعددة للمستأجرين المختلفين.

## 1. هيكل المجلد `packages/noble`

يتواجد الكود المصدري لكل إضافة داخل هذا المجلد، ويُعامل كل موديول وكأنه حزمة (Package) مستقلة من Laravel. الهيكل الداخلي النمطي والقياسي لأي موديول (مثل: `Account`) يكون كالتالي:

```text
packages/noble/Account
├── src
│   ├── Providers           # نقطة الدخول للحزمة (Service Providers) لتعريف المسارات والمناظر.
│   ├── Http
│   │   ├── Controllers     # المتحكمات الخاصة بالربط.
│   │   ├── Requests        # قواعد التحقق من الإدخالات الدخول (Form Requests).
│   ├── Models              # كائنات قاعدة البيانات المتعلقة بالمحاسبة مثلاً (Tax, AccountList, إلخ).
│   ├── Database
│   │   ├── Migrations      # جداول قاعدة البيانات الخاصة بالوحدة.
│   │   └── Seeders         # البيانات الافتراضية.
│   ├── Helpers             # دوال مساعدة خاصة كـ AccountUtility.php.
│   ├── Events              # أحداث مبرمجة مخصصة للوحدة.
│   └── Listeners           # المستمعون للأحداث للتعامل معها برمجياً.
└── composer.json           # ملف التعريف بالحزمة إن كانت مفصولة كلياً (اختياري احياناً).
```

## 2. آليات التفعيل والتعطيل التلقائي

كيف يعرف النظام ما إذا كان المستأجر / الشركة لديهم الصلاحية لدخول `Account` أم لا؟

- **PlanModuleCheck Middleware:**
  هذا الـ Middleware المرفق مع المنصة في مسار `app/Http/Middleware/PlanModuleCheck.php` يقوم باعتراض كل طلب (Request).
  يتم فحص الوحدة المرتبطة بالمسار المطلوب (يتم استخلاصها عبر الـ Web Routes)، ثم يراجع الـ Middleware اشتراك الشركة الحالية (Plan) وهل يتضمن هذا الـ Module بناءً على الجداول في قاعدة البيانات (`plan_modules`).

## 3. القائمة النهائية للوحدات المتاحة (Noble Ecosystem Modules)

يعمل النظام حالياً بـ **28 وحدة برمجية** متكاملة، مصنفة كالتالي:

### 🏦 المالية والمحاسبة (Finance & Accounting)
- **Account:** إدارة الحسابات، شجرة الحسابات، والميزانيات.
- **DoubleEntry:** نظام القيد المزدوج لضمان سلامة العمليات المالية.
- **BudgetPlanner:** التخطيط المالي وتتبع الميزانيات التقديرية.
- **Quotation:** إدارة عروض الأسعار والمقترحات المالية.

### 👥 الموارد البشرية (HRM Suite)
- **Hrm:** النواة الأساسية للموظفين، الرواتب، والإجازات.
- **Timesheet:** تتبع الحضور والانصراف وساعات العمل.
- **Recruitment:** نظام لطلبات التوظيف، الاختبارات، والمقابلات.
- **Performance:** تقييم أداء الموظفين ووضع الأهداف (KPIs).
- **Training:** إدارة الدورات التدريبية للموظفين وتط تطوير المهارات.

### 🏗️ إدارة العمليات والمشاريع (Operations & Projects)
- **Taskly:** نظام متطور لإدارة المشاريع والمهام (Kanban).
- **Contract:** إبرام العقود وتوثيق الاتفاقيات مع العملاء.
- **Goal:** وضع الأهداف الاستراتيجية وتتبع مستويات الإنجاز.

### 🛒 التجارة والمبيعات (Commerce & Sales)
- **Pos:** نظام نقطة البيع المتكامل مع المخازن.
- **ProductService:** إدارة المنتجات والخدمات والمخزون.
- **Lead:** إدارة العملاء المحتملين وفرص المبيعات (CRM).

### 🤖 الذكاء والاتصال (Intelligence & Communication)
- **AIAssistant:** رفيق الذكاء الاصطناعي المدمج (OpenAI/Anthropic).
- **SupportTicket:** نظام تذاكر الدعم الفني وخدمة العملاء.
- **Slack / Telegram / ZoomMeeting:** تكاملات التواصل الخارجية.
- **Twilio / Webhook:** أنظمة الرسائل والربط البرمجي.

### ⚙️ البنية التحتية (Infrastructure)
- **FormBuilder:** بناء النماذج المخصصة وجمع البيانات.
- **LandingPage:** إدارة الواجهة التعريفية للمنصة (SaaS Landing).
- **GoogleCaptcha:** حماية النظام من الهجمات الآلية.
- **Paypal / Stripe:** بوابات الدفع الإلكتروني للاشتراكات.

## 4. كيفية إنشاء موديول جديد

إذا رغب فريق التطوير لاحقاً في بناء وحدة مخصصة جديدة، مثلاً (Fleet Management):

1. قم بإنشاء مجلد البنية التحتية داخل `packages/noble/Fleet`.
2. انسخ البنية الأساسية كـ (Providers, Controllers).
3. سجل مزود الخدمة (Service Provider) بداخل قائمة المزودات في `bootstrap/providers.php` أو من خلال `composer.json` عبر قسم `autoload`.
4. أضف اسم الإضافة لقائمة `addons` في نظام باقات الاشتراك (Plans) من داخل قاعدة البيانات أو واجهة مدير المنصة المخصص (Super Admin) ليظهر في باقات الترقية للعملاء.
