# 🚀 دليل النشر والسيرفرات (Deployment & Server Guide)

يستعرض هذا الدليل الخطوات والإجراءات المعتمدة لرفع تطبيق Sukoun Albunyan إلى بيئة الإنتاج الفعلية، وخاصة عند التعامل مع خوادم (Hostinger) أو الخوادم السحابية بنظام (Linux).

## 1. الدليل السريع للإطلاق (Quick Launch Steps)

عليك دائماً تجهيز بيئة التطبيق محلياً قبل تصدير ملفات الـ (Build) لبيئة الإنتاج لتجنب تعطل التطبيق الحي. خطوات التجهيز هي:

1. تأكد من أن جميع الوحدات الإضافية (Modules) مفعلة وتعمل.
2. قم بتوليد ملفات الجافاسكربت و CSS المدمجة بالتوجه لمسار المشروع وتنفيذ الأمر:

```bash
npm run build
```

3. سيقوم الأمر المذكور بإنشاء مجلد `public/build` وتوليد الملفات اللازمة. سيتم استدعاؤها لاحقاً لتعمل بسرعة عالية جداً.

## 2. النشر على خوادم Hostinger

نظراً لأن بنية الاستضافة المشتركة أو المدارة لدى Hostinger تختلف قليلاً عن الخوادم الجاهزة بالكامل (VPS defaults)، يتم اتباع التالي:

- **أدوات الرفع الآلي:** تم تصميم سكربتات خاصة (متوفرة في مجلد `_scripts`) مثل `deploy_to_hostinger.ps1` (لبيئة الويندوز محلياً) ليقوم بضغط المشروع (`public_build.tar.gz` أو `noble_production_build.tar.gz`) ورفعها للخادم مباشرة باستخدام بروتوكول SFTP ثم فك الضغط داخلياً.
- **مسار الجذر المخصص (Document Root):**
  يجب التأكد من أن لوحة Hostinger تشير إلى مجلد `/public/` كالجذر الأساسي للموقع (`public_html` يجب أن ترتبط بمجلد `public` الداخلي لـ Laravel، أو يتم نقل الملفات العامة إلى الواجهة مع ضبط مسارات `index.php`).

## 3. التعامل مع ملفات PWA (Progressive Web App)

النظام مدعوم بالكامل ليتم تثبيته كتطبيق على الأجهزة المحمولة والكمبيوتر.

- تأكد أن ملف `public/manifest.json` متاح ولا يعطي أخطاء أثناء الرفع.
- تم ضبط علامات Meta الخاصة بدعم الأجهزة المحمولة ليكون التطبيق (Mobile Web App Capable). الروابط والأيقونات `favicon.ico` و صور التطبيق متصلة بمجلد `public/assets`.
- تأكد من صلاحيات المجلد `storage` و `bootstrap/cache` لتكون `775` للحفاظ على قدرة الوصول الصحيح بداخل الاستضافة:

```bash
chmod -R 775 storage bootstrap/cache
```

## 4. أداة الفحص والإصلاح الشامل بعد الرفع

بعد التأكد من استقرار الشفرة المصدرية على الخادم، توجد سلسلة من الأوامر لمعالجة الروابط وحفظها وتسريع الموقع، يمكن تنفيذها باستخدام اتصال SSH مع السيرفر:

```bash
composer install --optimize-autoloader --no-dev
php artisan optimize
php artisan migrate --force
```

إذا تعذر الدخول لـ SSH أو كان المستخدم يواجه أخطاء 500 بعد النشر، يُفضل استدعاء أحد ملفات الفحص המעקبة من المتصفح (مثل `_scripts/auto_fix_500.php` أو `_scripts/final_fix.php`) بشرط أن تكون مرفوعة على السيرفر مؤقتاً بصلاحيات آمنة. (ملاحظة: يجب حذفها فور الانتهاء من صيانتها حرصاً على الأمان).

## 5. الجدولة التلقائية (Cron Jobs)

لكي يعمل نظام التنبيهات وإرسال الفواتير الدورية وأتمتة `Workflow` بشكل ممتاز، يجب الانضمام للوحة الاستضافة وإضافة الـ Cron Job التالي بحيث يعمل كل دقيقة:

```bash
* * * * * cd /path-to-your-project && php artisan schedule:run >> /dev/null 2>&1
```

(استبدل `/path-to-your-project` بالمسار الفعلي للمشروع على الهوست).
